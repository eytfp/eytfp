# Java 主流框架时间线

   了解Java开发主流框架的**时间线**，理解框架的演进逻辑，避免只记名字却不懂背后的发展脉络。按**首次发布时间**梳理核心框架，同时标注“主流化时间”（实际大规模应用的阶段），并说明每个框架解决的核心问题，让你不仅知其然，还知其所以然。

### 一、Java主流框架时间线（按首次发布时间排序）

|框架/技术|首次发布时间|主流化时间|核心定位|解决的核心问题|
|---|---|---|---|---|
|EJB|1998年（J2EE 1.0）|2000-2005年|企业级Java组件模型|早期Java EE的分布式开发规范，试图统一企业级开发标准|
|Struts|2000年（Struts 1）|2002-2008年|Web MVC框架|解决Servlet/JSP开发的代码混乱问题，规范Web请求流程|
|Hibernate|2001年|2003-2010年|ORM框架|消除JDBC硬编码，实现“对象-关系”映射，简化数据库操作|
|Spring|2003年（Spring 1.0）|2005-至今|轻量级企业级框架|解决EJB的笨重问题，核心是IOC（解耦）+ AOP（切面编程）|
|MyBatis|2004年（原名iBatis）|2008-至今|半ORM框架|弥补Hibernate“黑盒操作”的不足，兼顾SQL灵活性和对象映射|
|Spring MVC|2004年（整合入Spring）|2008-2015年|Web MVC框架|替代Struts，与Spring无缝整合，成为Java Web MVC主流|
|Spring Boot|2014年（Spring Boot 1.0）|2016-至今|Spring简化框架|解决Spring配置繁琐问题，实现“自动配置、快速开发”|
|Spring Cloud|2015年|2017-2022年|微服务框架|基于Spring Boot，解决分布式微服务的注册、配置、熔断等问题|
|Spring Boot 3.x|2022年|2023-至今|新一代Spring Boot|基于Jakarta EE 9+，适配Java 17+，性能和安全性升级|
### 二、关键阶段解读（帮你理解演进逻辑）

#### 1. 早期阶段（2000年前）：无成熟框架，原生开发为主

- 开发方式：直接写Servlet/JSP+JDBC，代码耦合度极高（比如数据库连接、业务逻辑、页面渲染全混在一起）；

- 痛点：开发效率低、维护难，催生了第一批框架。

#### 2. 框架萌芽期（2000-2005年）：解决单点问题

- Struts：第一个主流Web MVC框架，把“请求接收-业务处理-页面返回”拆分开，规范Web开发；

- Hibernate：解决JDBC硬编码问题，让开发者用“面向对象”的方式操作数据库；

- Spring：核心是“轻量级”，替代笨重的EJB，通过IOC（控制反转）解耦代码，AOP（面向切面）解决日志、事务等通用问题。

#### 3. 框架整合期（2005-2015年）：SSM成为主流

- 核心组合：Spring + Spring MVC + MyBatis（SSM）；

- 背景：Struts逐渐被Spring MVC替代（更易整合Spring），Hibernate因SQL灵活性不足，被MyBatis分流；

- 痛点：配置文件繁多（XML配置），新手入门门槛高。

#### 4. 简化与云原生期（2015年至今）：Spring Boot/Cloud主导

- Spring Boot：核心是“约定大于配置”，自动配置Tomcat、数据库连接、依赖等，新手能快速搭建项目；

- Spring Cloud：基于Spring Boot实现微服务架构，适配互联网分布式场景；

- 趋势：框架越来越“轻量化”“自动化”，开发者更聚焦业务逻辑，而非配置。

### 三、新手学习建议（结合时间线）

1. **跳过“远古框架”**：EJB、Struts、原生Spring MVC（XML配置版）不用学，这些已被淘汰或边缘化；

2. **核心聚焦**：直接从**Spring Boot**入手（2016年后的主流），它整合了Spring、Spring MVC的核心功能，无需单独学习；

3. **配套学习**：Spring Boot + MyBatis 是当前企业开发的“标配组合”，掌握这两个就能应对80%的后端开发场景；

4. **进阶可选**：有基础后再了解Spring Cloud（微服务），但新手阶段不用碰。

### 总结

1. Java框架的演进核心是“从复杂到简化、从重型到轻型”，核心解决“代码耦合、配置繁琐、开发效率低”的问题；

2. 时间线关键节点：2003年Spring诞生（奠定基础）→ 2014年Spring Boot发布（简化开发）→ 至今Spring Boot主导；

3. 新手无需追溯所有框架，聚焦Spring Boot + MyBatis即可，这是当前最主流、最高效的学习路径。
