## 

###  什么是 Java 代码审计？

> 💡 **核心定义**：
>
> Java 代码审计（Code Audit）是指**通过阅读和分析 Java 源码**，找出程序中潜在的安全漏洞（如 SQL 注入、反序列化、XSS 等）和逻辑缺陷的过程。

- **审计方式**：
  - **白盒审计**：手握源码，从输入点（Contorller）追到触发点（Sink），或从敏感函数逆向追踪。
  - **动静结合**：静态看代码逻辑，动态通过 Debug 调试，观察内存中的变量变化。



### ☕ JDK 1.8 (Java Development Kit)

- **介绍**：Java 开发工具包。虽然 Java 更新到了高版本，但目前绝大多数企业老旧系统、开源漏洞靶场（如 WebGoat、VULHUB）依然深度依赖 **JDK 1.8（即 Java 8）**。它是审计生涯的立足之本。

- **安装步骤**：

  1. **下载**：前往 Oracle 官网或使用国内镜像（如华为云、清华源）下载 `jdk-8u411-windows-x64.exe`。

     [Java Archive Downloads - Java SE 8](https://www.oracle.com/java/technologies/javase/javase8-archive-downloads.html)

  2. **配置环境变量**：

     - 新建系统变量 `JAVA_HOME` = `C:\Program Files\Java\jdk1.8.0_xxx`
     - 修改 `Path` 变量，添加 `%JAVA_HOME%\bin`

  3. **验证**：打开命令行执行 `java -version`。

### 📦 Maven 3.8.8

- **介绍**：Java 项目构建和依赖管理工具。代码审计时，我们需要通过 Maven 的 `pom.xml` 文件快速查看项目引入了哪些第三方组件（如是否存在有漏洞的 `Log4j` 或 `Fastjson`）。

- **安装步骤**：

  1. **下载**：解压 `apache-maven-3.8.8-bin.zip` 到自定义目录。

  2. **配置环境变量**：

     - 新建系统变量 `MAVEN_HOME` = `你的解压路径`
     - 修改 `Path` 变量，添加 `%MAVEN_HOME%\bin`

  3. **换源（关键）**：修改 `conf/settings.xml`，在 `<mirrors>` 标签内加入阿里镜像加速下载：

     XML

     ```
     <mirror>
         <id>aliyunmaven</id>
         <mirrorOf>*</mirrorOf>
         <name>阿里云公共仓库</name>
         <url>https://maven.aliyun.com/repository/public</url>
     </mirror>
     ```

### 🗄️ MySQL 5.7.26

- **介绍**：关系型数据库。5.7 版本是代码审计中接触最多的数据库。了解它有助于在审计 **SQL 注入**漏洞时，理解预编译（`?` 占位符）和拼接的本质区别。

- **这个的安装过程多种多样，我是直接用的小皮面板，你也可以用其他的**

  

  

  

  ### 🐈 Tomcat 8.5.81

  *   **介绍**：经典的 Java Web 应用服务器（Servlet 容器）。审计传统的 `*.war` 包项目（如早期 Spring、Struts2、Servlet 项目）时，需要将代码部署到 Tomcat 中进行动态调试。
  *   **安装步骤**：
      1.  **下载**：下载 Core 版本的 `64-bit Windows zip`。
      2.  **解压即用**：直接解压到非中文路径。
      3.  **控制台乱码修复**：修改 `conf/logging.properties`，将 `java.util.logging.ConsoleHandler.encoding = UTF-8` 改为 `GBK`（防止 Windows 命令行下启动报错全是乱码）。
      4.  **启动**：双击 `bin/startup.bat`。

  

  

  ### 🎯 审计利器：IntelliJ IDEA 2023.1.2

  *   **介绍**：当之无愧的 Java 审计第一神器。2023.1.2 版本**对全局搜索、依赖关系图（Dependency Analyzer）以及反编译插件**的支持非常稳定，完美适配各种漏洞挖掘插件（如 *FindBugs*、*CheckStyle*、*Glowworm*）。

  ### ⚡ 审计向配置与安装

  1.  **安装**：下载安装包，一路 Next。
  2.  **与环境联动（核心配置）**：
      *   **配置 JDK**：`File -> Project Structure -> SDKs`，导入你的 JDK 1.8。
      *   **配置 Maven**：`Settings -> Build, Execution, Deployment -> Build Tools -> Maven`：
          *   **Maven home path**: 选择你的 Maven 3.8.8 路径。
          *   **User settings file**: 选择你的 `settings.xml` 路径。
